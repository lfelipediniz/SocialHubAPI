    Social Features Documentation

    Overview

    The Social Features API provides comprehensive social interaction capabilities including likes, comments, shares, and user relationships. These features enhance user engagement and create a vibrant social platform.

    Base URLs

    • Posts Social Features: http://localhost:8000/careers/
    • User Social Features: http://localhost:8000/users/

    Authentication

    • Authentication: Use Authorization: Bearer <token> header

    Post Interactions

    Likes System

    Like a Post

    POST /careers/{id}/like/

    Like a specific post.

    Request: 

    curl -X POST "http://localhost:8000/careers/1/like/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    URL Parameters:

    • id (required) - The unique identifier of the post to like

    Response: 201 Created 

    {
  "message": "Post liked successfully"
}

    Unlike a Post

    DELETE /careers/{id}/unlike/

    Remove like from a post.

    Request: 

    curl -X DELETE "http://localhost:8000/careers/1/unlike/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    URL Parameters:

    • id (required) - The unique identifier of the post to unlike

    Response: 204 No Content

    List Post Likes

    GET /careers/{id}/likes/

    Get all likes for a specific post with pagination.

    Request: 

    curl -X GET "http://localhost:8000/careers/1/likes/?batch_size=10&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    URL Parameters:

    • id (required) - The unique identifier of the post

    Query Parameters:

    • batch_size (optional) - Number of likes per batch (default: 10, max: 100)

    • batch_number (optional) - Batch number starting from 0 (default: 0)

    Response: 

    {
  "message": "Likes retrieved successfully",
  "likes": [
    {
      "id": 1,
      "user": "alice",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "batch_info": {
    "current_batch": 0,
    "batch_size": 10,
    "total_likes": 25,
    "total_batches": 3,
    "likes_in_current_batch": 10
  }
}

    Comments System

    Add Comment

    POST /careers/{id}/comment/

    Add a comment to a post.

    Request: 

    curl -X POST "http://localhost:8000/careers/1/comment/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "This is a great post! I really enjoyed reading it."
  }'

    URL Parameters:

    • id (required) - The unique identifier of the post to comment on

    Request Body Parameters:

    • content (required) - The comment text content

    Response: 201 Created 

    {
  "id": 1,
  "user": "alice",
  "content": "This is a great post! I really enjoyed reading it.",
  "created_at": "2024-01-15T10:30:00Z"
}

    Create Shared Post

    POST /careers/{id}/share-post/

    Create a new post that shares another post with your own commentary.

    Request: 

    curl -X POST "http://localhost:8000/careers/1/share-post/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My thoughts on this post",
    "content": "I found this post really interesting because..."
  }'

    URL Parameters:

    • id (required) - The unique identifier of the original post to share

    Request Body Parameters:

    • title (required) - The title for the new shared post

    • content (required) - Your thoughts/commentary about the original post

    Response: 201 Created 

    {
  "id": 2,
  "username": "alice",
  "title": "My thoughts on this post",
  "content": "I found this post really interesting because...",
  "created_datetime": "2024-01-15T10:30:00Z",
  "original_post": 1,
  "post_type": "share"
}

    User Relationships

    Following System

    Follow User

    POST /users/follow/

    Follow another user to see their posts in your feed.

    Request: 

    curl -X POST "http://localhost:8000/users/follow/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "following": 2
  }'

    Request Body Parameters:

    • following (required) - ID of the user to follow

    Response: 201 Created 

    {
  "id": 1,
  "follower": 1,
  "following": 2,
  "follower_username": "alice",
  "following_username": "bob",
  "created_at": "2024-01-15T10:30:00Z"
}

    Unfollow User

    DELETE /users/{username}/unfollow/

    Stop following a user.

    Request: 

    curl -X DELETE "http://localhost:8000/users/bob/unfollow/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    URL Parameters:

    • username (required) - The username of the user to unfollow

    Response: 200 OK 

    {
  "message": "Stopped following bob"
}

    Feed System

    Personalized Feed

    GET /users/me/feed/

    Get personalized feed with posts from users you follow, plus your own posts. Uses the same batch system as other endpoints for consistency.

    Request: 

    curl -X GET "http://localhost:8000/users/me/feed/?batch_size=20&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Query Parameters:

    • batch_size (optional) - Number of posts per batch (default: 20, max: 100)

    • batch_number (optional) - Batch number starting from 0 (default: 0)

    Response: 

    {
  "message": "Feed retrieved successfully",
  "posts": [
    {
      "id": 1,
      "username": "bob",
      "title": "New Post from Bob",
      "content": "This is a post from someone I follow",
      "created_datetime": "2024-01-15T10:30:00Z"
    },
    {
      "id": 2,
      "username": "alice",
      "title": "My Own Post",
      "content": "This is my own post",
      "created_datetime": "2024-01-15T09:30:00Z"
    }
  ],
  "batch_info": {
    "current_batch": 0,
    "batch_size": 20,
    "total_posts": 50,
    "total_batches": 3,
    "posts_in_current_batch": 20
  }
}

    Get comprehensive statistics for the current user.

    Request: 

    curl -X GET "http://localhost:8000/users/me/stats/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Response: 

    {
  "posts_count": 15,
  "followers_count": 42,
  "following_count": 28,
  "likes_received": 156,
  "comments_received": 89
}

    Error Handling

    Status Codes

    • 200 OK - Successful operation

    • 201 Created - Resource created successfully

    • 204 No Content - Successful operation with no content

    • 400 Bad Request - Invalid request data

    • 401 Unauthorized - Authentication required

    • 403 Forbidden - Insufficient permissions

    • 404 Not Found - Resource not found

    • 409 Conflict - Resource already exists (e.g., already following)

    Error Response Format

    {
  "error": "Error message",
  "details": "Additional error details"
}

    Examples

    Like a Post

    What to send:

    • Replace <your_token> with your actual authentication token

    • Replace 1 in the URL with the post ID you want to like

    What you'll receive:

    • Status: 201 Created

    • Response: {"message": "Post liked successfully"}

    Command: 

    curl -X POST "http://localhost:8000/careers/1/like/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Add a Comment

    What to send:

    • Replace <your_token> with your actual authentication token

    • Replace 1 in the URL with the post ID you want to comment on

    • Replace "Great post!" with your comment content

    What you'll receive:

    • Status: 201 Created

    • Response: The created comment with id, user, content, and created_at

    Command: 

    curl -X POST "http://localhost:8000/careers/1/comment/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{"content": "Great post!"}'

    Get Personalized Feed
