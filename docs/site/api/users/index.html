    Users API Documentation

    Overview

    The Users API provides comprehensive user management functionality including registration, authentication, profile management, and social features like following/followers system.

    Base URL

    http://localhost:8000/users/

    Authentication Modes

    JWT Authentication

    • Secure token-based authentication
    • Use Authorization: Bearer <token> header
    • Tokens expire after 60 minutes
    • Refresh tokens available for 7 days

    Endpoints

    Authentication

    Register User

    POST /users/register/

    Register a new user account.

    Request: 

    curl -X POST "http://localhost:8000/users/register/" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "email": "alice@example.com",
    "password": "password123",
    "password_confirm": "password123",
    "first_name": "Alice",
    "last_name": "Smith",
    "bio": "Software developer passionate about APIs"
  }'

    Request Body Parameters:

    • username (required) - Unique username for the account

    • email (required) - Valid email address

    • password (required) - Account password

    • password_confirm (required) - Password confirmation (must match password)

    • first_name (optional) - User's first name

    • last_name (optional) - User's last name

    • bio (optional) - User's biography/description

    Response: 201 Created 

    {
  "user": {
    "id": 1,
    "username": "alice",
    "email": "alice@example.com",
    "first_name": "Alice",
    "last_name": "Smith",
    "bio": "Software developer passionate about APIs",
    "created_at": "2024-01-15T10:30:00Z"
  },
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

    Login

    POST /users/login/

    Authenticate user and get access tokens.

    Request: 

    curl -X POST "http://localhost:8000/users/login/" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "password": "password123"
  }'

    Request Body Parameters:

    • username (required) - Username or email for authentication

    • password (required) - User's password

    Response: 200 OK 

    {
  "user": {
    "id": 1,
    "username": "alice",
    "email": "alice@example.com",
    "first_name": "Alice",
    "last_name": "Smith"
  },
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

    Refresh Token

    POST /users/token/refresh/

    Get new access token using refresh token.

    Request: 

    curl -X POST "http://localhost:8000/users/token/refresh/" \
  -H "Content-Type: application/json" \
  -d '{
    "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }'

    Request Body Parameters:

    • refresh (required) - Valid refresh token

    Response: 200 OK 

    {
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

    Logout

    POST /users/logout/

    Logout user and blacklist refresh token.

    Request: 

    curl -X POST "http://localhost:8000/users/logout/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }'

    Request Body Parameters:

    • refresh (required) - Valid refresh token to blacklist

    Response: 200 OK 

    {
  "message": "Successfully logged out"
}

    User Management

    List Users

    GET /users/

    Get paginated list of all users with search and filtering using batch system.

    Request: 

    curl -X GET "http://localhost:8000/users/?search=alice&batch_size=10&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Query Parameters:

    • search (optional) - Search in username, first_name, last_name, bio

    • first_name (optional) - Filter by first name (contains)

    • last_name (optional) - Filter by last name (contains)

    • ordering (optional) - Sort by: username, created_at, first_name, last_name

    • batch_size (optional) - Number of users per batch (default: 10, max: 100)

    • batch_number (optional) - Batch number starting from 0 (default: 0)

    Response: 

    {
  "message": "Users retrieved successfully",
  "users": [
    {
      "id": 1,
      "username": "alice",
      "first_name": "Alice",
      "last_name": "Smith",
      "avatar": "https://example.com/avatar.jpg",
      "is_following": false
    }
  ],
  "batch_info": {
    "current_batch": 0,
    "batch_size": 10,
    "total_users": 25,
    "total_batches": 3,
    "users_in_current_batch": 10
  }
}

    Response:

  "username": "alice",
  "first_name": "Alice",
  "last_name": "Smith",
  "bio": "Software developer passionate about APIs",
  "avatar": "https://example.com/avatar.jpg",
  "created_at": "2024-01-15T10:30:00Z",
  "posts_count": 15,
  "followers_count": 42,
  "following_count": 28
}

    Update My Profile

    PATCH /users/me/update/

    Update current user's profile information.

    Request: 

    curl -X PATCH "http://localhost:8000/users/me/update/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Alice",
    "last_name": "Johnson",
    "bio": "Updated bio",
    "avatar": "https://example.com/new-avatar.jpg",
    "email": "alice.johnson@example.com"
  }'

    Request Body Parameters:

    • first_name (optional) - User's first name

    • last_name (optional) - User's last name

    • bio (optional) - User's biography/description

    • avatar (optional) - URL to user's avatar image

    • email (optional) - User's email address

    Response: 200 OK 

    {
  "id": 1,
  "username": "alice",
  "email": "alice.johnson@example.com",
  "first_name": "Alice",
  "last_name": "Johnson",
  "bio": "Updated bio",
  "avatar": "https://example.com/new-avatar.jpg",
  "updated_at": "2024-01-15T11:30:00Z"
}

    Social Features

    Follow User

    POST /users/follow/

    Follow another user.

    Request: 

    curl -X POST "http://localhost:8000/users/follow/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "following": 2
  }'

    Request Body Parameters:

    • following (required) - ID of the user to follow

    Response: 201 Created 

    {
  "id": 1,
  "follower": 1,
  "following": 2,
  "follower_username": "alice",
  "following_username": "bob",
  "created_at": "2024-01-15T10:30:00Z"
}

    Unfollow User

    DELETE /users/{username}/unfollow/

    Stop following a user.

    Request: 

    curl -X DELETE "http://localhost:8000/users/bob/unfollow/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    URL Parameters:

    • username (required) - The username of the user to unfollow

    Response: 200 OK 

    {
  "message": "Stopped following bob"
}

    User Statistics and Feed

    Get My Statistics

    GET /users/me/stats/

    Get current user's statistics.

    Request: 

    curl -X GET "http://localhost:8000/users/me/stats/" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Response: 

    {
  "posts_count": 15,
  "followers_count": 42,
  "following_count": 28,
  "likes_received": 156,
  "comments_received": 89
}

    Feed Functionality

    Note: The personalized feed functionality (posts from followed users + your own posts) is available through the main posts API:

    • GET /careers/ - Get all posts (use frontend filtering to show only followed users)
    • GET /users/me/stats/ - Get your following/followers information
    • GET /users/<username>/followers/ - Get list of followers
    • GET /users/<username>/following/ - Get list of following

    This approach is more efficient and avoids route duplication.

    Error Handling

    Status Codes

    • 200 OK - Successful operation

    • 201 Created - Resource created successfully

    • 204 No Content - Successful operation with no content

    • 400 Bad Request - Invalid request data

    • 401 Unauthorized - Authentication required

    • 403 Forbidden - Insufficient permissions

    • 404 Not Found - Resource not found

    • 409 Conflict - Resource already exists

    Error Response Format

    {
  "error": "Error message",
  "details": "Additional error details"
}

    Examples

    Register New User

    What to send:

    • Replace "alice" with your desired username

    • Replace "alice@example.com" with your email address

    • Replace "password123" with your password (use it twice for password_confirm)

    • Replace "Alice" and "Smith" with your first and last name

    What you'll receive:

    • Status: 201 Created

    • Response: User object with access and refresh tokens

    Command: 

    curl -X POST "http://localhost:8000/users/register/" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "email": "alice@example.com",
    "password": "password123",
    "password_confirm": "password123",
    "first_name": "Alice",
    "last_name": "Smith"
  }'

    Login and Get Token

    What to send:

    • Replace "alice" with your username

    • Replace "password123" with your password

    What you'll receive:

    • Status: 200 OK

    • Response: User object with access and refresh tokens

    Command: 

    TOKEN=$(curl -X POST "http://localhost:8000/users/login/" \
  -H "Content-Type: application/json" \
  -d '{"username": "alice", "password": "password123"}' | jq -r '.access')

    Update Profile

    What to send:

    • Replace $TOKEN with your actual access token

    • Replace "Software developer passionate about APIs" with your bio

    What you'll receive:

    • Status: 200 OK

    • Response: Updated user profile

    Command: 

    curl -X PATCH "http://localhost:8000/users/me/update/" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"bio": "Software developer passionate about APIs"}'

    Follow Another User

    What to send:

    • Replace $TOKEN with your actual access token

    • Replace 2 with the ID of the user you want to follow

    What you'll receive:

    • Status: 201 Created

    • Response: Follow relationship object

    Command: 

    curl -X POST "http://localhost:8000/users/follow/" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"following": 2}'

    List Users with Pagination

    What to send:

    • Replace 5 with desired number of users per page (optional, default: 10)

    • Replace 0 with the page number you want (optional, default: 0)

    What you'll receive:

    • Status: 200 OK

    • Response: Users list with pagination info

    Command: 

    curl -X GET "http://localhost:8000/users/?batch_size=5&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Search Users

    What to send:

    • Replace "alice" with your search term

    • Replace 10 with desired number of users per page (optional, default: 10)

    • Replace 0 with the page number you want (optional, default: 0)

    What you'll receive:

    • Status: 200 OK

    • Response: Filtered users list with pagination info

    Command: 

    curl -X GET "http://localhost:8000/users/?search=alice&batch_size=10&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    Filter Users by Name

    What to send:

    • Replace "Alice" with the first name to filter by

    • Replace "Smith" with the last name to filter by

    What you'll receive:

    • Status: 200 OK

    • Response: Filtered users list

    Command: 

    curl -X GET "http://localhost:8000/users/?first_name=Alice&last_name=Smith" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"

    List Users with Custom Ordering

    What to send:

    • Replace "created_at" with desired ordering field (username, created_at, first_name, last_name)

    • Replace 10 with desired number of users per page (optional, default: 10)

    • Replace 0 with the page number you want (optional, default: 0)

    What you'll receive:

    • Status: 200 OK

    • Response: Sorted users list with pagination info

    Command: 

    curl -X GET "http://localhost:8000/users/?ordering=created_at&batch_size=10&batch_number=0" \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json"
