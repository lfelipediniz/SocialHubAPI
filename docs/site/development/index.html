    Development Guide

    This guide covers development setup and configuration for the SocialHubAPI project.

    Development Setup

    Prerequisites

    • Python 3.12+
    • pip and virtualenv
    • Git
    • PostgreSQL (for production)

    Environment Setup

    1. Clone and Setup

    git clone https://github.com/lfelipediniz/SocialHubAPI
cd socialhubapi
python -m venv venv
source venv/bin/activate  # Linux/macOS
# or venv\Scripts\activate  # Windows

    2. Install Dependencies

    pip install -r requirements.txt

    3. Environment Configuration

    # Copy environment template
cp env.example .env

# Edit .env with your development settings

    4. Database Setup

    # Run migrations
python manage.py migrate

# Create superuser (optional)
python manage.py createsuperuser

    5. Start Development Server

    python manage.py runserver 0.0.0.0:8000

    Project Structure

    socialhubapi/
├── posts/                    # Posts app
│   ├── models.py            # Post, Like, Comment, Share models
│   ├── views.py             # API views
│   ├── serializers.py       # Data serializers
│   ├── urls.py              # URL routing
│   ├── permissions.py       # Custom permissions
│   └── tests.py             # Unit tests
├── users/                    # Users app
│   ├── models.py            # User, Follow models
│   ├── views.py             # User management views
│   ├── serializers.py       # User serializers
│   ├── authentication.py    # Custom authentication
│   └── tests.py             # User tests
├── socialhubapi/            # Django project settings
│   ├── settings.py          # Main settings
│   ├── urls.py              # Root URL configuration
│   └── wsgi.py              # WSGI configuration
├── docs/                    # Documentation
│   ├── docs/                # MkDocs content
│   └── mkdocs.yml           # MkDocs configuration
└── requirements.txt         # Python dependencies

    Configuration

    Environment Variables

    Required Variables

    # Django Settings
DJANGO_SECRET_KEY=your-secret-key-here
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1

# Database
DATABASE_URL=sqlite:///db.sqlite3  # Development
# DATABASE_URL=postgresql://user:pass@localhost/db  # Production

# Authentication
JWT_SECRET_KEY=your-jwt-secret-key-here

# CORS
DJANGO_CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000

    Optional Variables

    # JWT Settings (Production only)
JWT_SECRET_KEY=your-jwt-secret-key

# Email Settings
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password

    Settings Configuration

    The project uses python-decouple for environment variable management:

    # socialhubapi/settings.py
from decouple import config

# Basic Django settings
SECRET_KEY = config('DJANGO_SECRET_KEY', default='django-insecure-...')
DEBUG = config('DJANGO_DEBUG', default=True, cast=bool)
ALLOWED_HOSTS = config('DJANGO_ALLOWED_HOSTS', default='localhost,127.0.0.1').split(',')

# Authentication mode
# JWT configuration is automatically loaded from environment

    Database Management

    Development Database

    # SQLite (default for development)
DATABASE_URL=sqlite:///db.sqlite3

    Production Database

    # PostgreSQL
DATABASE_URL=postgresql://user:password@localhost:5432/socialhubapi

    Database Operations

    # Reset database
rm db.sqlite3
python manage.py migrate

# Create superuser
python manage.py createsuperuser

# Dump data
python manage.py dumpdata > fixtures/backup.json

# Load data
python manage.py loaddata fixtures/backup.json

    Documentation

    MkDocs Setup

    # Install MkDocs
pip install mkdocs mkdocs-material

# Serve documentation locally
cd docs
mkdocs serve

# Build documentation
mkdocs build

    Adding Documentation

    1. Create new .md file in docs/docs/
    2. Add to navigation in mkdocs.yml
    3. Update cross-references
    4. Test with mkdocs serve

    Deployment

    Production Checklist

    Security

    • [ ] Set DEBUG=False
    • [ ] Use strong SECRET_KEY
    • [ ] Configure ALLOWED_HOSTS
    • [ ] Use HTTPS
    • [ ] Set secure CORS origins
    • [ ] Use production database

    Performance

    • [ ] Enable database connection pooling
    • [ ] Configure caching (Redis)
    • [ ] Use CDN for static files
    • [ ] Enable gzip compression
    • [ ] Set up monitoring

    Environment

    # Production environment variables
DJANGO_DEBUG=False
DJANGO_SECRET_KEY=your-production-secret-key
DJANGO_ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
DATABASE_URL=postgresql://user:pass@localhost/socialhubapi
AUTHENTICATION_REQUIRED=True

    GitHub Pages Deployment

    Setup GitHub Pages

    # Install gh-pages plugin
pip install mkdocs-gh-deploy

# Deploy to GitHub Pages
mkdocs gh-deploy

    Configuration

    The documentation is automatically deployed to GitHub Pages when you push to the main branch.

    Common Issues

    Import Errors

    # Make sure virtual environment is activated
source venv/bin/activate
pip install -r requirements.txt

    Database Issues

    # Reset migrations
rm posts/migrations/0*.py
rm users/migrations/0*.py
python manage.py makemigrations
python manage.py migrate

    Permission Errors

    # Check file permissions
chmod +x manage.py
chmod -R 755 .
